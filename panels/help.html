Help

You can use this extension to control the permissions of websites. For example, enabling sandbox features, enabling or disabling scripts, stylesheets, fonts and choosing which third parties can provide subresources.

The first thing you probably want to do fine tune the default rules, you can do that by choosing the default ruleset in the Options page.

Report

The primary page for interacting with matrix³. This page allows you to precisely control the permissions of the active website.

Session and Dynamic Rules

When you toggle a checkbox in the Report page, matrix³ creates a session rule for that host. Session rules are ephemeral, they only last until you close the browser. Once you're happy with the result, click Commit to convert the session rule into a dynamic rule, which persists across browser restarts.

If you change your mind about an edit, click Abandon to discard the session rule and restore the previously committed dynamic rule. To go the other way -- make a persistent rule temporary again -- click Uncommit, which demotes the dynamic rule back to a session rule (so it disappears when you close the browser). Reset removes every rule for the host -- session and dynamic both.

Violations

When a page action is blocked by the active policy, matrix³ records the source and highlights the relevant checkbox. Click Unblock to allow every reported source at once instead of ticking them one by one.

Server Suggested Policy

The Server Suggested Policy shows the policy the server sent for the current origin. Accept replaces your current selection with the server's policy; Merge instead adds the server's sources to your existing selection.

Please note, not all directives can be enabled. In particular, nonces and hashes are imported but not enabled to prevent 'unsafe-inline' from being disabled. You can still manually enable them.

Groups

A group is a named bundle of origins you frequently want to trust together, for example a group containing the CDNs you allow on most sites. Define groups in the Groups page, then in the Report page pick the group and click Trust to allow all its origins for the current host at once. Click Untrust to remove them.

The Ignore group is special: any origin you add to it is hidden from the Report page entirely. Use this for noisy trackers that you would never allow and don't want spamming the source list.

Options

Always Show Advanced Directives

Some Content Security Policy directives like worker-src, child-src, manifest-src, and object-src are rarely needed. By default, matrix³ keeps their columns hidden in the Report page and only reveals a column when one of its cells has a violation or a checked source. Enable this option to keep all the advanced columns visible at all times.

Show Icon Badges

When enabled, matrix³ marks the toolbar icon with a red ! badge whenever a CSP violation is reported for the active tab, so you know at a glance there is something to look at on the Report page. The badge clears on the next navigation.
